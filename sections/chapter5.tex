\chapter{系统实现与关键模块}

\section{关键实现文件与模块关系}
本系统采用“平台入口统一调度、算法脚本相对独立”的实现方式。平台交互入口为 \texttt{src/app.py}，主要负责 Gradio 界面构建、用户参数接收、默认路径识别、模型权重加载和外部脚本调度；数据预处理由 \texttt{src/preprocess\_mri.py} 完成；二维图像生成训练和推理复用 \texttt{src/stylegan2} 目录中的 StyleGAN2-ADA 代码；面向三维重建的点云初始化由 \texttt{src/init\_pcd.py} 完成。该结构使界面层和算法层保持相对解耦，便于在后续继续接入真实 3DGS 训练流程。

\section{MRI 数据预处理模块实现}
MRI 数据预处理模块主要由 \texttt{preprocess\_mri.py} 实现。脚本通过命令行参数接收输入目录、输出目录和目标切片尺寸，并使用 \texttt{nibabel} 读取 \texttt{.nii} 或 \texttt{.nii.gz} 格式的 NIfTI 体数据。考虑到脑部 MRI 体数据两端通常包含较多背景区域，系统沿轴向提取中间区间的二维切片，默认保留深度方向约 15\% 到 85\% 的范围，以减少头顶、颈部和空白区域对训练集质量的影响。

在灰度处理方面，系统没有直接使用全局最小值和最大值归一化，而是采用 2\% 到 98\% 百分位截断归一化。该方法能够削弱极端亮点和异常噪声对整体灰度分布的影响，使不同病例、不同切片之间具有更稳定的灰度范围。归一化后的切片被转换为 \texttt{uint8} 类型，并通过双三次插值缩放到目标尺寸。为了避免空白切片进入训练集，系统进一步计算切片平均灰度、标准差和非零像素比例，只有满足有效信息量要求的切片才会被保存为 PNG 文件。

预处理完成后，平台可调用 StyleGAN2 的 \texttt{dataset\_tool.py} 将 PNG 切片目录打包为训练所需的 zip 数据集。根据当前训练配置文件，项目使用的训练集路径为 \texttt{data/processed/mri128.zip}，最大样本规模为 60815 张，图像分辨率为 $128\times128$。这一处理流程保证了原始医学体数据能够转换为 StyleGAN2 可直接消费的二维图像训练集。

\section{StyleGAN2 训练与切片生成实现}
StyleGAN2 训练功能由平台入口 \texttt{app.py} 进行封装。平台首先在 \texttt{data/processed} 目录中查找可用训练数据，优先识别 \texttt{mri256.zip}、\texttt{mri128.zip} 和 \texttt{mri\_slices.zip} 等文件。找到训练集后，系统通过 \texttt{subprocess.Popen} 启动 StyleGAN2 训练脚本，并将输出目录、数据路径、GPU 数量、训练 kimg、batch size、学习率和快照保存频率等参数传入训练进程。长时间训练任务在后台运行，避免阻塞 Gradio 主界面。

切片生成模块使用训练权重中的 \texttt{G\_ema} 生成器进行推理。平台会自动查找最新的 \texttt{network-snapshot-*.pkl} 权重文件，也允许用户在界面中手动指定模型路径。推理时，系统根据用户输入的随机种子采样潜变量，并结合截断参数 \texttt{truncation\_psi} 控制生成结果的稳定性和多样性。生成器输出的张量先从 $[-1,1]$ 范围转换到 $[0,255]$，再保存为 PNG 图像。当前项目已经形成可加载的权重文件 \texttt{network-snapshot-010000.pkl}，并生成示例切片 \texttt{seed0123.png}。

该模块的实现重点不只是完成一次图像生成，而是将训练结果接入平台工作流。用户可以在界面中选择生成数量、随机种子和截断参数，平台按序保存生成结果，并以画廊形式展示。这样既方便后续论文插图整理，也便于答辩演示中说明模型输入、推理过程和输出结果之间的关系。

\section{连续切片序列与点云初始化实现}
连续切片序列生成由 \texttt{app.py} 中的 \texttt{generate\_sequence} 函数实现。系统分别根据两个随机种子生成潜变量 $z_a$ 和 $z_b$，再对二者进行线性插值，得到一组连续变化的潜变量序列。每个插值潜变量输入 StyleGAN2 生成器后得到一张 MRI 切片，最终按顺序保存为 \texttt{slice\_0000.png}、\texttt{slice\_0001.png} 等文件。该设计能够为后续点云初始化和 2D/3D 对比展示提供连续输入。

点云初始化由 \texttt{init\_pcd.py} 实现。脚本读取连续切片目录中的 PNG 图像，将灰度值大于阈值 10 的像素转换为空间点。对于第 $z$ 张切片中的像素坐标 $(x,y)$，系统将其映射为三维点 $(x,y,z\cdot spacing)$，其中 \texttt{spacing} 表示切片间距。像素灰度值被归一化后映射为 RGB 颜色，所有点和颜色最终以 ASCII PLY 格式写入文件。当前项目已生成 \texttt{results/3d\_model\_test/init\_point\_cloud.ply}，包含 477426 个顶点，文件大小约 13 MB。

需要强调的是，该步骤属于“面向 3DGS 的输入准备”和“初始点云构建”，不是完整的 3DGS 训练结果。完整 3DGS 还需要在初始点云基础上进一步优化三维高斯的位置、尺度、旋转、不透明度和颜色等参数，并结合渲染损失进行迭代训练。因此，本文在当前阶段将该模块定位为二维生成结果向三维表示过渡的工程接口。

\section{平台前端与交互功能实现}
平台前端基于 Gradio Blocks 构建，采用多标签页组织主要功能。数据预处理页用于设置原始数据目录、输出目录和目标切片尺寸；模型训练页用于设置学习率、批大小和训练 kimg；医学切片生成页用于加载模型权重并批量生成 MRI 切片；2D/3D 对比页同时提供连续切片序列生成和点云初始化入口。各模块的输入参数和输出日志均显示在同一平台中，便于用户理解完整流程。

在工程实现上，平台使用当前 Python 解释器路径启动外部脚本，避免界面进程和子进程使用不同 Python 环境导致依赖不一致。对于训练任务，系统使用后台进程启动；对于预处理、推理生成和点云初始化等相对短任务，系统直接返回日志文本、图像结果或输出文件路径。同时，平台封装了 \texttt{\_find\_latest\_pkl} 和 \texttt{\_find\_training\_data\_zip} 等辅助函数，用于自动定位最新权重和训练数据，降低用户手动配置路径的成本。

\section{AI 助手功能实现}
AI 助手集成在平台界面右下角，以悬浮按钮和聊天面板形式出现。系统预设了面向 MRI 医学图像生成平台的提示词，使助手回答时围绕 StyleGAN2、3DGS、MRI 图像处理和平台使用展开。平台通过兼容 OpenAI SDK 的调用方式接入外部大模型服务，并支持在不同模型提供商之间切换。

该功能的定位是教学辅助和参数解释，而不是平台核心算法依赖。用户可以询问截断参数、随机种子、点云初始化、3DGS 后续训练和 MRI 切片生成等问题。由于 AI 助手依赖外部模型接口，其可用性会受到网络和 API 服务状态影响，因此系统的数据处理、模型训练、图像生成和点云初始化功能均保持独立运行。这样即使 AI 助手不可用，平台的核心毕业设计功能仍能完成。
