\chapter{系统实现与关键模块}

\section{MRI 数据预处理模块实现}
MRI 数据预处理模块主要由 \texttt{preprocess\_mri.py} 实现。该脚本通过命令行参数接收输入目录、输出目录、目标尺寸和最大处理文件数，并使用 nibabel 读取 NIfTI 文件。对于每个 MRI 体数据，系统沿轴向提取二维切片，并按照设定比例保留中间区域。

在灰度处理方面，系统没有直接使用全局最小值和最大值归一化，而是采用 2\% 到 98\% 百分位归一化。这样可以减少异常亮点或极端噪声对整体灰度范围的影响。归一化后的切片被转换为 \texttt{uint8} 类型，并使用双三次插值缩放到目标尺寸。系统进一步通过均值、标准差和非零像素比例判断切片是否有效，过滤空白切片和信息量不足的切片。

预处理完成后，平台调用 StyleGAN2 的 \texttt{dataset\_tool.py} 将 PNG 切片目录打包为训练所需的 zip 数据集。当前项目中已形成 \texttt{mri128.zip} 和 \texttt{mri\_slices.zip} 等训练数据文件，训练配置显示模型使用的训练集最大规模为 60815 张 $128\times128$ 切片。

\section{StyleGAN2 训练与切片生成实现}
StyleGAN2 训练和推理功能主要由平台入口 \texttt{app.py} 调度。训练入口函数会自动在 \texttt{data/processed} 目录下查找可用训练数据，优先识别 \texttt{mri256.zip}、\texttt{mri128.zip} 和 \texttt{mri\_slices.zip} 等文件。找到训练数据后，平台通过子进程调用 StyleGAN2 训练脚本，并传入输出目录、数据路径、GPU 数量、训练迭代量、批大小、学习率和快照保存频率等参数。

模型训练完成后，系统使用 \texttt{.pkl} 权重文件中的 \texttt{G\_ema} 生成器进行推理。切片生成函数根据用户输入的随机种子采样潜变量，并在无梯度环境下调用生成器得到图像张量。随后系统将生成张量从 $[-1,1]$ 范围转换到 $[0,255]$，再保存为 PNG 图像。当前项目已有训练完成的权重文件 \texttt{network-snapshot-010000.pkl}，文件大小约 283 MB，并已生成示例切片 \texttt{seed0123.png}。

平台生成模块支持批量生成。对于生成数量为 $n$ 的任务，系统从基础随机种子开始递增生成 $n$ 张图像，并将输出保存到 \texttt{results/generated\_slices} 目录。该设计便于用户快速获得多张生成结果，并将其作为论文图像展示材料。

\section{连续切片序列与点云初始化实现}
连续切片序列生成由 \texttt{app.py} 中的 \texttt{generate\_sequence} 函数实现。系统分别根据两个随机种子生成潜变量 $z_a$ 和 $z_b$，然后对二者进行线性插值，得到一组连续变化的潜变量序列。每个插值潜变量输入 StyleGAN2 生成器后得到一张 MRI 切片，最终按序号保存为 \texttt{slice\_0000.png}、\texttt{slice\_0001.png} 等文件。

点云初始化由 \texttt{init\_pcd.py} 实现。该脚本读取连续切片目录中的 PNG 图像，将灰度值大于阈值的像素转换为空间点。对于第 $z$ 张切片中的像素坐标 $(x,y)$，系统将其映射为三维点 $(x,y,z\cdot spacing)$，其中 \texttt{spacing} 表示切片间距。像素灰度值被归一化后映射为 RGB 颜色。所有点和颜色最终以 ASCII PLY 格式写入文件。

该实现的优点是逻辑清晰、依赖较少，能够快速从二维切片序列得到可查看的三维点云。其不足是尚未进行高斯基元参数优化，也没有加入医学体数据的真实物理间距和相机约束。因此在论文中，本文将该部分表述为“初始点云构建”或“面向 3DGS 的输入准备”，不将其夸大为完整 3DGS 重建结果。

\section{平台前端与交互功能实现}
平台前端由 Gradio Blocks 构建。系统在一个页面中组织多个标签页，每个标签页对应一个主要流程。数据预处理页使用文本框接收路径，使用滑块设置目标尺寸，使用按钮触发预处理；模型训练页使用数值输入和下拉框设置训练参数；切片生成页使用图像画廊展示生成结果；2D/3D 对比页同时提供序列生成和点云初始化入口。

平台在执行外部脚本时使用当前 Python 解释器路径，避免界面进程和子进程使用不同 Python 环境造成依赖不一致。对于长时间训练任务，系统使用后台进程启动，避免训练过程阻塞 Gradio 界面。对于推理生成和点云初始化等相对短任务，系统直接返回输出图片、日志文本和文件路径。

平台还提供默认路径自动填充功能。例如，系统会自动查找最新的 \texttt{network-snapshot} 权重文件，作为切片生成和序列生成模块的默认模型路径。这样用户在完成训练后无需手动查找权重文件，即可继续进行推理和展示。

\section{AI 助手功能实现}
AI 助手功能集成在平台界面右下角，以悬浮按钮和聊天面板形式出现。系统预设了面向 MRI 医学图像生成平台的提示词，使助手回答时围绕 StyleGAN2、3DGS、MRI 图像处理和平台使用展开。平台通过兼容 OpenAI SDK 的调用方式接入外部大模型服务，并允许在不同模型提供商之间切换。

AI 助手的定位是教学辅助和参数解释。用户可以询问截断参数、随机种子、3DGS 点云初始化和 MRI 切片生成等问题。由于该功能依赖外部模型接口，其可用性受到网络和 API 服务状态影响。因此系统核心功能不依赖 AI 助手，数据处理、模型训练和图像生成均可独立运行。
