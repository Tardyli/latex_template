\chapter{系统实现与关键模块}

\section{关键实现文件与模块关系}
本系统采用“统一平台入口调度、算法模块独立实现”的代码组织方式。平台入口文件为 \texttt{src/app.py}，主要承担 Gradio 界面构建、用户参数接收、默认路径识别、模型权重加载和外部脚本调度等任务。MRI 数据预处理由 \texttt{src/preprocess\_mri.py} 实现，二维生成训练与推理复用 StyleGAN2-ADA 工程，点云初始化则由独立脚本完成。

3DGS 相关代码与训练产物分别保存在三维框架目录和结果目录中。这样的划分使界面层、二维生成模块和三维重建模块之间保持相对独立，便于调试和扩展。

\section{MRI 数据预处理模块实现}
MRI 数据预处理模块由 \texttt{preprocess\_mri.py} 实现。脚本接收输入目录、输出目录和目标尺寸等参数，并借助 \texttt{nibabel} 读取 \texttt{.nii} 或 \texttt{.nii.gz} 格式的 NIfTI 文件。考虑到脑部 MRI 体数据在两端常包含较大背景或低信息区域，系统默认沿轴向提取中间约 15\% 到 85\% 的切片，以减少头顶部、颈部和空白切片对训练样本分布的影响。

灰度处理阶段，系统采用 2\% 到 98\% 百分位截断，而不是直接使用最小值和最大值进行拉伸。这样可以降低异常亮点、噪声和局部极值对整体灰度范围的影响。归一化后的切片被转换为 \texttt{uint8} 类型，并通过双三次插值缩放到目标分辨率。为了控制训练集质量，脚本继续统计切片平均灰度、标准差和非零像素比例，只保存满足信息量要求的切片。

预处理完成后，平台可以调用 StyleGAN2 的 \texttt{dataset\_tool.py} 将 PNG 切片目录打包为训练数据。本次训练使用的数据集文件为 \texttt{mri128.zip}，训练集最大规模为 60815 张，图像分辨率为 $128\times128$。该流程实现了从医学体数据到二维生成模型训练样本的转换。

\section{StyleGAN2 训练与切片生成实现}
StyleGAN2 训练由平台入口 \texttt{app.py} 封装。平台会在训练数据目录中查找 zip 数据包，并优先使用本项目配置的 \texttt{mri128.zip}。当训练任务启动后，系统通过 \texttt{subprocess.Popen} 调用 StyleGAN2 训练脚本，将输出目录、数据路径、GPU 数量、训练 kimg、batch size、学习率和快照保存频率等参数传入子进程。由于训练耗时较长，后台进程可以避免 Gradio 主界面被阻塞。

切片生成模块使用训练权重中的 \texttt{G\_ema} 生成器进行推理。平台既能自动查找最新网络快照，也允许用户手动指定权重路径。推理阶段，系统根据随机种子生成潜变量，并结合截断参数 \texttt{truncation\_psi} 调节输出的稳定性与多样性。生成器输出张量从 $[-1,1]$ 转换到 $[0,255]$ 后保存为 PNG 图像。本项目已经形成编号为 010000 的模型快照，并生成了可用于论文展示的示例切片。

该模块的重点在于把训练产物真正接入平台，而不是只在脚本层面完成一次生成。用户可以通过界面设置生成数量、随机种子和截断参数，系统则负责保存图像、返回路径并以画廊形式展示结果。这样既方便论文插图整理，也便于答辩时说明参数、模型和输出之间的对应关系。

\section{连续切片序列与点云初始化实现}
连续切片序列由 \texttt{app.py} 中的 \texttt{generate\_sequence} 函数生成。系统分别根据两个随机种子构造潜变量 $z_a$ 和 $z_b$，再对二者进行线性插值，得到一组连续变化的潜变量。每个插值潜变量输入 StyleGAN2 生成器后输出一张 MRI 切片，并按照 \texttt{slice\_0000.png}、\texttt{slice\_0001.png} 等规则保存。该功能为 2D/3D 对比展示提供了连续输入。

点云初始化由 \texttt{init\_pcd.py} 完成。脚本读取连续切片目录中的 PNG 图像，将灰度值大于阈值 10 的像素转换为空间点。对于第 $z$ 张切片中的像素坐标 $(x,y)$，系统将其映射为三维点 $(x,y,z\cdot spacing)$，其中 \texttt{spacing} 表示切片间距。像素灰度值归一化后被写入 RGB 颜色字段，所有点最终以 ASCII PLY 格式保存。本项目已生成 \texttt{results/3d\_model\_test/init\_point\_cloud.ply}，该文件包含 477426 个顶点，大小约 13 MB。

通过这一模块，二维切片不再只是独立图像，而是被组织为空间坐标和灰度颜色共同描述的三维数据。该点云既能作为 3DGS 训练的初始化输入，也能作为平台展示二维生成结果向三维表达过渡的中间证据。

\section{3DGS 三维重建模块实现}
为了让三维重建流程与平台功能保持一致，项目保留了 3DGS 框架目录，并围绕 MRI 切片序列、初始点云、相机参数、训练日志和输出点云设计统一的数据接口。该实现路线对应“二维生成结果组织为三维输入，再通过 3DGS 优化形成高斯表达”的工程目标，使平台能够展示初始点云，也能归档训练后的三维重建结果。

3DGS 模块输入主要包括三类数据：第一类是连续 MRI 切片序列，用作渲染监督；第二类是根据切片灰度和空间位置初始化得到的 PLY 点云；第三类是描述切片投影关系的相机参数文件。普通 3DGS 场景通常借助 COLMAP 估计相机位姿，而 MRI 切片序列具有规则层间关系，因此系统将切片索引、像素间距和层间距转换为规则相机参数。当前训练目录中的 \texttt{cameras.json} 共包含 180 个相机条目，图像分辨率为 $256\times256$，焦距参数 $fx$ 与 $fy$ 均为 426.67。

训练实现上，系统使用 180 张连续 MRI 切片和 300000 个初始化点构建 3DGS 输入数据，并将结果保存到 3DGS 主训练目录。训练过程会记录相机参数、配置参数、训练日志、checkpoint、渲染图像、参考切片和迭代点云。第 30000 轮输出的 PLY 文件头部显示包含 572073 个高斯点，同时保存 \texttt{chkpnt30000.pth}，用于记录训练后的模型状态。

从平台实现角度看，三维重建模块接收切片目录和输出目录等核心参数，并返回训练日志、点云路径和渲染结果路径。系统先完成点云初始化，再组织训练迭代次数、输出目录、相机参数和渲染结果目录等信息。平台沿用日志框和三维展示区域，使三维结果能够在同一交互界面中完成查看、下载和归档。

本文将该部分定义为 3DGS 三维重建模块，是因为项目已经以真实切片序列、PLY 点云文件、训练日志、checkpoint 和渲染截图为基础，完成了从二维切片组织到三维高斯表达结果的工程闭环。相关训练指标和渲染效果将在第 6 章进一步分析。

\section{平台前端与交互功能实现}
平台前端基于 Gradio Blocks 构建，并通过多标签页组织核心功能。数据预处理页用于设置原始数据目录、输出目录和目标尺寸；模型训练页用于设置学习率、批大小和训练 kimg；医学切片生成页用于加载权重并批量生成 MRI 切片；2D/3D 对比页用于生成连续切片序列、启动点云初始化并展示三维相关输出。各模块的输入参数和输出日志集中显示，便于用户按流程完成实验。

工程实现中，平台使用当前运行环境中的 Python 解释器启动外部脚本，减少界面进程和子进程依赖不一致带来的问题。对于训练任务，系统使用后台进程；对于预处理、推理生成和点云初始化等耗时较短的任务，系统直接返回日志文本、图像结果或输出路径。同时，平台封装了 \texttt{\_find\_latest\_pkl} 和 \texttt{\_find\_training\_data\_zip} 等辅助函数，用于自动定位最新权重和训练数据，降低用户手动配置成本。

\section{AI 助手功能实现}
AI 助手以悬浮按钮和聊天面板形式集成在平台右下角。系统设置了面向 MRI 医学图像生成平台的提示词，使回答内容尽量围绕 StyleGAN2、3DGS、MRI 图像处理和平台使用流程展开。平台通过兼容 OpenAI SDK 的调用方式接入外部大模型服务，并支持在不同模型提供商之间切换。

该功能的定位是教学辅助和参数解释，而不是核心算法依赖。用户可以询问截断参数、随机种子、点云初始化、3DGS 三维表达和 MRI 切片生成等问题。由于 AI 助手依赖外部接口，其可用性会受到网络和 API 服务影响，因此系统的数据处理、模型训练、图像生成和点云初始化功能均保持独立运行。即使 AI 助手不可用，平台的主要毕业设计功能仍能正常完成。
